## Critical Architectural Rules (must know before writing any code)

### Google Drive URLs
- Always use `https://lh3.googleusercontent.com/d/{FILE_ID}=w2000` format (Google CDN)
- The gdrive-adapter.ts generates this automatically on every upload
- BackgroundGallery.tsx has an old fallback to `drive.google.com/thumbnail` ‚Äî known inconsistency, low priority

### Storage Sync Pattern (universal ‚Äî no exceptions)
- Every table that stores files MUST have: `storage_provider`, `storage_path`, `storage_url`, `gdrive_file_id`
- Deletion order is ALWAYS: delete from storage first, then delete from DB
- DB triggers queue deletions; cron job processes every 5 minutes
- `docs/` folder has been moved to `archived_docs/` ‚Äî storage sync pattern is fully live in DB triggers

### Database Constraint (RESOLVED)
- `composites_angled_shot_id_background_id_key` ‚Äî this constraint NO LONGER EXISTS in live DB (was dropped)
- Current composites unique constraint is only `UNIQUE(category_id, slug)` ‚Äî A/B testing is unblocked

### Known DB Quirk (not breaking, leave alone)
- `product_images` table has no `category_id` column ‚Äî intentional, join through `products` table instead

### Template System (fully built)
- API routes exist, DB schema ready, one template per (category, format) enforced
- All layer positions stored as percentages (0‚Äì100) for scale independence
- Default template data in `src/types/templates.ts`
- Safe zones must be respected in composites and final asset generation

## Planned Feature: "Bring Your Own Copy" (CSV ingestion)
**Context:** Requested by Moritz (CDO, Sunday Natural) after Jutta (CMO) delivered finished ad creatives with pre-written hooks/captions. Moritz wants AdForge to accept external copy as an alternative to AI-generated copy, and overlay it onto composite images the same way.

**What to build:**
1. **CSV ingestion endpoint** ‚Äî accept a CSV upload with columns:
   - `asset_name` ‚Äî e.g. "Cycle Komplex ‚Äì Hook 1"
   - `composite_image_url` ‚Äî Google Drive URL of the composite
   - `hook` / `headline` ‚Äî short overlay text
   - `caption_c1`, `caption_c2`, `caption_c3` ‚Äî ad captions (not overlaid, used for Meta)
   - `format` ‚Äî 1:1 / 4:5 / 9:16
2. **Text overlay engine** ‚Äî use `sharp` (server-side Node.js) to render hook/headline text onto composite images with brand font, size, and placement guardrails
3. **Final ad export** ‚Äî save finished images (text baked in) to Google Drive; output CSV with asset names + Drive URLs for manual Meta upload
4. **UI** ‚Äî new section in the category workflow (Phase 6) with two modes:
   - "AI Copy" ‚Äî current flow (copy generated by tool)
   - "Import Copy" ‚Äî CSV upload mode

**Notes:**
- Phases 6 & 7 are currently stubs (`501 Not Implemented`) ‚Äî this feature fills them
- Text baking onto images is NEW for both copy modes (currently copy is stored as text only, not rendered onto images)
- Meta upload is NOT in scope ‚Äî manual upload from Drive remains the process
- Font/style/placement guardrails need to be defined as part of this work (not yet defined in codebase)
